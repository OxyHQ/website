# Tools

The Oxy MCP server registers **92 tools**, grouped here by the CMS surface they operate on. All tool inputs are validated with [Zod](https://zod.dev) on the server, so calls with bad shapes return a clear error before they hit the database.

Every tool returns a single text content block whose body is the JSON-stringified result. Errors come back with `isError: true` and the stack trace in the text body.

<Callout variant="info" title="Reading the schemas">
  Each input shape below is the TypeScript form Claude sees when picking arguments. `?` marks optional fields. `&#x7c;` marks a discriminated union. `string ObjectId` means a 24-character Mongo `_id` string.
</Callout>

## Pages

Mongo collection: `pages`. CMS-managed marketing pages with composable section blocks.

### `list_pages`

List every page slug.

```ts
input: {}
output: Array<{ slug: string; title: string }>
```

### `get_page`

Fetch a single page by slug, including its sections array.

```ts
input: { slug: string }
output: Page
```

### `upsert_page`

Create or update a page. Sections is a flexible array of `{ type, heading?, subheading?, content?, items?, order? }` blocks.

```ts
input: {
  slug: string
  title: string
  description?: string
  sections?: Array<Section>
  promptPhrases?: string[]
}
```

## Navigation

Mongo collection: `navigations`. Top-level header dropdowns.

### `get_navigation`

Returns every dropdown sorted by `order`.

```ts
input: {}
output: Array<NavDropdown>
```

### `replace_navigation`

Replaces the entire navigation with the supplied array. Destructive: existing dropdowns are deleted first.

```ts
input: {
  items: Array<{
    label: string
    items: Array<{ title, description?, href, icon?, section?, showGrid? }>
    sidePanel?: { heading, links: Array<{ label, href }> }
    order?: number
  }>
}
```

## Newsroom

Mongo collection: `newsroomposts`.

### `list_posts`

Filterable, paginated list sorted by `publishedAt` desc.

```ts
input: {
  category?: string         // e.g. "Company", "Research", "Product", "Safety", "Engineering", "Security"
  tag?: string
  featured?: boolean
  status?: 'draft' | 'published'
  search?: string           // matches title or resume
  limit?: number            // default 20
  page?: number             // default 1
}
output: { posts: Post[], total, page, pages }
```

### `get_post`

```ts
input: { slug: string }
output: Post  // with coverImage and ogImage populated
```

### `create_post`

Slug auto-generated from title if omitted; collision suffix appended on conflict.

```ts
input: {
  title: string
  slug?: string
  resume?: string
  description?: string
  content?: string          // Markdown
  coverImage: string        // Media _id, required
  imageAlt?: string
  tags?: string[]
  categories?: string[]
  featured?: boolean
  colorPrimary?: string
  colorSecondary?: string
  dark?: boolean
  status?: 'draft' | 'published'
  oxyUserId?: string
  metaTitle?: string
  ogImage?: string          // Media _id
  publishedAt?: string      // ISO date
}
```

### `update_post`

Patch by slug. Pass `newSlug` to rename. Only provided fields change.

```ts
input: {
  slug: string                // current slug
  newSlug?: string
  ...all create_post fields, optional
}
```

### `delete_post`

```ts
input: { slug: string }
output: { deleted: true, slug }
```

### `search_posts`

Lightweight search across `title` and `resume`.

```ts
input: { query: string, limit?: number /* default 10 */ }
output: Post[]
```

### `get_post_with_media`

Same as `get_post`, but always populates `coverImage` and `ogImage`.

```ts
input: { slug: string }
output: Post
```

## Media library

Mongo collection: `media`. Backed by DigitalOcean Spaces (S3-compatible).

### `list_media`

```ts
input: {
  search?: string
  type?: 'image' | 'video' | 'document'
  tag?: string
  folder?: string
  limit?: number, page?: number
}
output: { items, total, page, pages }
```

### `get_media`

```ts
input: { id: string }
output: Media
```

### `update_media`

```ts
input: { id: string, alt?: string, tags?: string[], folder?: string }
```

### `delete_media`

Deletes the S3 object **and** its thumbnails, then removes the Mongo document.

```ts
input: { id: string }
output: { deleted: true, id }
```

### `upload_image`

Downloads an image from a URL, uploads to Spaces, generates thumbnails, creates a Media document. Returns the full Media object.

```ts
input: {
  url: string
  filename?: string         // derived from URL if omitted
  folder?: string           // subfolder in oxy-website/, default "images"
  alt?: string
  tags?: string[]
}
output: Media
```

### `upload_and_set_post_cover`

One-step variant: download → upload → create Media → set as `coverImage` on the named post.

```ts
input: {
  postSlug: string
  imageUrl: string
  filename?: string
  alt?: string
}
output: { media: Media, post: Post }
```

### `upload_and_set_team_avatar`

Same flow for team member avatars.

```ts
input: {
  memberSlug: string
  imageUrl: string
  filename?: string
  alt?: string
}
output: { media: Media, member: TeamMember }
```

### `bulk_upload_post_covers`

Batch variant — one network round-trip per post, results collected.

```ts
input: { posts: Array<{ slug: string, imageUrl: string, alt?: string }> }
output: Array<{ slug, status: 'ok' | 'error', mediaId?, error? }>
```

### `debug_upload_test`

Diagnoses each step of the upload pipeline (download → buffer → S3 → Media). Use this to debug `upload_image` failures. Cleans up after itself.

```ts
input: { url: string }
output: { success: boolean, steps: string[] }
```

## Locales

Mongo collection: `locales`. The set of languages the site supports.

### `list_locales`

```ts

```

### `create_locale`

Setting `isDefault: true` clears the flag on every other locale first.

```ts
input: {
  code: string              // BCP-47, e.g. "ca-ES"
  slug?: string             // defaults to lower-cased code
  name: string              // English name, e.g. "Catalan"
  nativeName: string        // e.g. "Català"
  isDefault?: boolean
  enabled?: boolean         // default true
  order?: number
}
```

### `update_locale`

```ts
input: { code: string, ...patch fields }
```

### `delete_locale`

Refuses to delete the default locale. Also deletes every translation for that locale.

```ts
input: { code: string }
output: { deleted: true, code, translationsRemoved: number }
```

## MCP tokens

Mongo collection: `mcptokens`. Self-management of the same tokens that authenticate MCP calls.

### `list_mcp_tokens`

Token hashes are **never** returned.

```ts
input: {}
output: Array<{ _id, name, createdBy, createdAt, lastUsedAt, expiresAt, revoked }>
```

### `create_mcp_token`

Returns the raw token **once**. Save it immediately — there is no way to recover it.

```ts
input: { name: string, expiresAt?: string /* ISO date */ }
output: { _id, name, token: string /* raw secret */, createdAt, expiresAt }
```

### `revoke_mcp_token`

Sets `revoked: true`. The token will fail validation on the very next request.

```ts
input: { id: string }
output: { revoked: true, id }
```
