# Oxy MCP Server

The Oxy MCP server exposes the [oxy.so](https://oxy.so) website content management system as a set of tools that any [Model Context Protocol](https://modelcontextprotocol.io) client can call — most commonly **Claude Desktop** and **Claude Code**, but also any agent built on the official Anthropic SDK or a third-party MCP client.

## What you get

Instead of clicking through the admin panel, you can ask Claude things like:

- "Publish a newsroom post titled *Oxy raises Series B*, tag it Company, and use the press kit image at this URL."
- "List the help articles in the *Getting started* category and rewrite their summaries to be one sentence."
- "Create a referral code `ALEX-2026`, ambassador type, status active."
- "Show me every product on the technologies grid and flag the ones missing a logo."

Each natural-language request is routed by Claude to one or more MCP tools, which talk to MongoDB and DigitalOcean Spaces on the website's behalf.

## What it is, briefly

The [Model Context Protocol](https://modelcontextprotocol.io) is an open standard from Anthropic for letting LLMs call external tools and read external data. An MCP server speaks JSON-RPC 2.0 over a transport (stdio, HTTP, or SSE) and advertises a list of tools, resources, and prompts to any connected client.

The Oxy MCP server is a [`McpServer`](https://github.com/modelcontextprotocol/typescript-sdk) instance built with `@modelcontextprotocol/sdk`, mounted at `POST /mcp` on `api.oxy.so` over the **Streamable HTTP** transport.

## Who it's for

- **Operators of oxy.so** running their CMS through Claude instead of (or alongside) the admin UI.
- **Internal automations** that bulk-import posts, sync changelog entries from GitHub releases, or refresh translations.
- **AI app developers** building agents that need to read or modify Oxy website content programmatically.

It is **not** the same as the Oxy REST API. The REST API at `api.oxy.so` powers public Oxy apps (identity, files, billing, etc.); the MCP server is a thin layer over the website's internal Mongo models that ships with the CMS itself.

## What it does not expose

- No **resources** (`resources/list`) — the server only registers `tools`. All content is fetched via tool calls.
- No **prompts** (`prompts/list`) — the server advertises no canned prompts.
- No public endpoints — every request requires a valid MCP token (see [Authentication](/developers/docs/mcp/authentication)).

## Where to go next

1. **Get connected.** Follow the [Quickstart](/developers/docs/mcp/quickstart) to wire Claude Desktop or Claude Code to the server in about five minutes.
2. **Generate a token.** Read [Authentication](/developers/docs/mcp/authentication) to mint a token from the admin console.
3. **Browse the tools.** [Tools](/developers/docs/mcp/tools) lists every callable tool with its input schema and an example response.
4. **Try a real task.** [Examples](/developers/docs/mcp/examples) shows complete workflows — "publish a post with an AI-generated cover image", "spin up a referral campaign", "translate the navbar into Catalan".
