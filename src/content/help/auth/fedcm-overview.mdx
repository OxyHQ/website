Oxy apps use a **device-first, zero-cookie** session model. Each web origin (or native app) stores its own `{ deviceId, deviceSecret }` pair. The client mints short-lived bearer tokens by presenting that secret to the Oxy API — possession of the secret is the proof.

## Why it matters

Cross-domain cookies, FedCM, and SSO redirect bounces were removed ecosystem-wide. Device-first auth keeps each app's session scoped to its own origin while still letting users switch accounts in-app through the shared `OxyAccountDialog`.

## How it works

1. The user signs in once inside the app (`OxyAccountDialog` — QR, password, or Commons handoff).
2. The SDK persists `deviceId` + `deviceSecret` for that origin.
3. On reload, `OxyProvider` cold-boots: mints a fresh access token from the stored secret via `POST /session/device/token`.
4. Private API calls wait on `useAuth().canUsePrivateApi` until the token is ready.

<Callout type="info" title="No session cookies">
  There is no `oxy_device` cookie, no refresh-token family, and no FedCM surface. Device secrets live in first-party storage only.
</Callout>

## Building on top

Wrap your app with `OxyProvider` from `@oxyhq/services` and register your app's OAuth `clientId`:

```tsx
import { OxyProvider, useAuth } from '@oxyhq/services'

function App() {
  return (
    <OxyProvider baseURL="https://api.oxy.so" clientId={import.meta.env.VITE_OXY_CLIENT_ID}>
      <YourApp />
    </OxyProvider>
  )
}

function SignInButton() {
  const { isAuthenticated, signIn } = useAuth()
  if (isAuthenticated) return null
  return <button type="button" onClick={() => signIn()}>Sign in with Oxy</button>
}
```

See the [services SDK docs](/developers/docs/services) for the full integration guide.
