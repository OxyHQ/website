[FedCM](https://developers.google.com/privacy-sandbox/3pcd/fedcm) is the browser API for federated sign-in without third-party cookies. Oxy is a FedCM **identity provider** — every Oxy app uses it to sign you in without the redirect-bounce pattern.

## Why it matters

Before FedCM, third-party cookies were how sites knew you were signed in elsewhere. Browsers are deprecating them. FedCM gives us a privacy-preserving replacement: the browser mediates the consent and only shares the minimum needed.

## How Oxy uses it

1. You sign in to Accounts once.
2. Any other Oxy app (Inbox, Console, Mention) opens with a FedCM call to the Oxy IdP.
3. The browser shows a one-tap consent sheet ("Continue as Nate Isern").
4. Your session is provisioned without a single redirect.

<Callout type="info" title="No third-party cookies needed">
  Oxy works without third-party cookies. FedCM is the mechanism.
</Callout>

## Building on top

If you're building an app that wants to sign users in with their Oxy ID, you can use the FedCM flow directly. The auth-sdk handles all the wiring.

```ts
import { WebOxyProvider, useWebOxy } from '@oxyhq/auth'

function App() {
  return (
    <WebOxyProvider baseURL="https://api.oxy.so">
      <YourApp />
    </WebOxyProvider>
  )
}
```

See the [auth-sdk docs](/developers/docs/auth) for the full integration.
